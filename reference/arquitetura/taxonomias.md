# TAXONOMIAS

## Função
Este arquivo define como o sistema organiza seus elementos em categorias.

Ele existe para:
- evitar mistura de tipos de informação;
- manter separação clara entre camadas;
- facilitar navegação, leitura e manutenção;
- permitir crescimento sem perda de estrutura.

Taxonomia não é estética.  
É organização com função operacional.

---

## Princípios

- categorias devem refletir função, não tema;
- cada elemento deve ter um lugar natural;
- evitar duplicação entre categorias;
- manter separação entre estrutural, operacional e memória;
- não criar categorias sem necessidade real.

---

## Camadas principais do sistema

### 1. Core
Função:
- identidade estrutural do sistema

Inclui:
- SOUL
- USER
- MEMORY (núcleo conceitual)
- PRINCIPIOS
- EVOLUTION

---

### 2. System
Função:
- operação viva do sistema

Inclui:
- SYSTEM
- NOW
- PRIORITIES
- SPRINT
- OPEN-LOOPS
- DASHBOARD-STATUS

---

### 3. Areas
Função:
- frentes contínuas da vida e operação

Inclui:
- vida
- recursos
- profissional
- operação

Observação:
- não são projetos
- não têm fim definido

---

### 4. Projects
Função:
- iniciativas com objetivo específico

Inclui:
- Excellentia
- Moisés
- Mente Conexa
- Painel Caio Leite
- OpenClaw
- outros

Observação:
- têm estado (status)
- evoluem ao longo do tempo

---

### 5. Reference
Função:
- regras, padrões e integrações

Inclui:
- arquitetura (taxonomias, convenções, critérios, fontes de verdade)
- integrações
- pessoas

---

### 6. Memory
Função:
- memória auxiliar e histórica viva

Inclui:
- decisões
- lições
- registros
- diário

---

### 7. Archive
Função:
- material não operacional

Inclui:
- versões antigas
- projetos encerrados
- histórico

---

## Regras de classificação

- algo estrutural → core ou system  
- algo contínuo → areas  
- algo com objetivo → projects  
- regra ou padrão → reference  
- registro de memória → memory  
- sem uso ativo → archive  

---

## Erros comuns a evitar

- transformar projeto em área  
- misturar memória com operação  
- duplicar informação entre camadas  
- criar subcategorias sem necessidade  
- usar taxonomia como decoração
