# CONVENÇÕES

## Função
Este arquivo define padrões de escrita, nomeação e organização dos arquivos.

Ele existe para:
- manter consistência no sistema;
- facilitar leitura e manutenção;
- evitar ambiguidade e duplicação;
- permitir expansão sem perda de clareza.

Convenção não é rigidez.  
É redução de atrito.

---

## Princípios

- clareza acima de sofisticação;
- consistência acima de preferência pontual;
- simplicidade acima de detalhamento excessivo;
- padrão deve ajudar, não atrapalhar;
- exceções são permitidas quando justificadas.

---

## Nome de arquivos

### Regra geral
- nomes simples e diretos
- usar minúsculas
- usar hífen para separação

Exemplos:
- `financas.md`
- `producao-intelectual.md`
- `anuncios-meta.md`

---

### Evitar
- nomes excessivamente longos
- nomes ambíguos
- variações desnecessárias para o mesmo conceito

---

## Estrutura interna dos arquivos

Padrão recomendado:

1. Função  
2. Princípios (quando aplicável)  
3. Conteúdo principal  
4. Observações  
5. Relações com o sistema  
6. Última atualização confiável  

---

## Escrita

- evitar linguagem genérica
- evitar prolixidade desnecessária
- evitar simplificação que perde sentido
- priorizar clareza com densidade
- escrever para uso real, não para aparência

---

## Links internos

- usar links relativos entre arquivos
- conectar arquivos relevantes
- evitar isolamento de informação

Exemplo:
- `[[SYSTEM]]`
- `[[PRIORITIES]]`
- `[[../projects/excellentia/excellentia]]`

---

## Atualização

- atualizar em vez de duplicar
- evitar coexistência de versões conflitantes
- refletir estado real do sistema
- registrar mudanças relevantes
